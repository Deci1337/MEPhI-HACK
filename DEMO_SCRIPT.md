# Demo Script — HexTeam Messenger

## Устройства

| Устройство | Платформа | Порт |
|-----------|-----------|------|
| Device A | Windows | 45680 |
| Device B | Android | 45680 |
| Device C | Windows | 45681 |

Все устройства — в одной локальной сети (Wi-Fi или Android hotspot).

---

## Сценарий 1: Discovery + Direct Chat (2 мин)

1. Запустить A и B
2. Подождать 5–10 сек — B появляется в списке Peers у A (UDP discovery)
3. Кликнуть на B в списке → он выбирается как активный пир
4. Отправить сообщение с A: ввести текст → Send
5. На B отображается сообщение. Статус у A меняется: Sent → Delivered
6. Отправить ответ с B → A видит сообщение

**Что показывает жюри:** автоматическое обнаружение устройств, двусторонний чат, подтверждение доставки.

---

## Сценарий 2: Relay A → B → C (2 мин)

1. Запустить все три устройства
2. A подключён к B, C подключён к B — A и C напрямую не подключены
3. С A отправить сообщение C (ввести nodeId C как получателя)
4. Сообщение уходит: A → B (relay) → C
5. В TechLog на C: `hop=1` — подтверждает relay-прохождение

**Что показывает жюри:** multihop forwarding через промежуточный узел.

---

## Сценарий 3: File Transfer (2 мин)

1. На A нажать File → выбрать любой файл
2. Наблюдать прогресс в строке FileTransfer
3. После завершения — в чате появится `[File] filename SHA256 OK`
4. На B файл сохранён в папку HexTeamReceived

**Что показывает жюри:** chunked transfer с проверкой целостности SHA256.

---

## Сценарий 4: Voice Call (1.5 мин)

1. На A выбрать B, нажать Call
2. На B появляется баннер входящего звонка → Accept
3. Оба говорят — голос идёт через UDP в реальном времени
4. Нажать Hang Up для завершения

**Что показывает жюри:** real-time voice через UDP.

---

## Сценарий 5: Voice Messages (1 мин)

1. На A выбрать B, нажать REC → говорить → STOP
2. Голосовое появляется в чате у B с кнопкой Play
3. Нажать Play — воспроизвести

**Что показывает жюри:** асинхронные голосовые сообщения как элементы чата.

---

## Сценарий 6: Metrics (30 сек)

1. Показать нижнюю панель метрик: RTT, LOSS, QUALITY, RETRIES
2. Объяснить: обновляется каждые 3 секунды через ping/pong

---

## Fallback — если Discovery не работает

Ввести IP:port вручную:
1. В поле IP:port в нижней части левой панели — ввести адрес
2. Нажать Connect
3. Пир добавляется и запоминается в истории

---

## Порядок для демо

```
A запускается первым (порт 45680)
C запускается с HEX_TCP_PORT=45681
B (Android) запускается последним
```

Если A и C в одной сети — они обнаружат друг друга автоматически.
